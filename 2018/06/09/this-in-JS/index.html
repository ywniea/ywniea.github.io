this in JS

Jun 9, 2018 | JavaScript

this的绑定完全取决于函数的调用位置。

方法一：默认绑定

默认绑定发生在单独调用函数的情况下，会将this 绑定到全局对象(not strict mode)或者是undefined(strict mode)。

在严格模式下运行会绑定this到undefined。
但是在严格模式下调用并不会影响默认绑定，依然会绑定到全局对象。

方法二：隐式绑定

调用位置有上下文对象的，会使用这个上下文对象obj1作为上下文来引用函数，将this指向obj1。

并且，只有最近的一层上下文对象会起作用。如代码中的obj3.obj2.obj1.foo()起作用的是obj1.

隐式绑定的函数丢失绑定对象，则采用默认绑定，会根据是否是严格模式将this绑定到全局对象或者是undefined。

方法三：显示绑定

使用bind, apply, call

使用api。例如Array.prototype.forEach接受第二个参数作为上下文对象。

方法四：new

  1. 创建一个全新的对象
  2. 将这个对象.__proto__属性指向构造函数.prototype
  3. 运行构造函数，一般是给这个全新的对象添加属性
  4. 如果没有显示返回其他的对象，就会隐式返回这个新创建的对象。

优先级

new > 显示绑定bind, apply, call > 隐式绑定obj.foo() > 默认绑定

绑定例外

情况一：
目标函数不需要、不使用this的时候，传null/undefined，会导致默认绑定到全局对象或者undefined。

此时可以传一个空对象Object.create(null)

情况二：

赋值表达式obj2.foo = obj1.foo的返回值是目标函数的引用，因此在这里调用位置是foo()而不是obj1.foo() 或者obj2.foo()

箭头函数

=> 相当于 var self = this;
箭头函数的绑定无法修改。一旦箭头函数的this绑定了，new, apply, call, bind等都不能修改。

箭头函数不使用上面四种方法来决定this，而是使用外层作用域来决定this，会继承外层函数调用的this绑定。

如果不理解可自行将=> 改写为 var self = this;然后在接下来的代码中用self。

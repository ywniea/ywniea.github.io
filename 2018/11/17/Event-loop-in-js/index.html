Event loop in js

Nov 17, 2018 | JavaScript

Event loop

JS是单线程的。只有一个调用栈call stack，每次只能做一件事情。主线程会首先去执行同步代码。遇到异步代码就会挂起，然后继续执行后面的同步代码。
当异步代码返回结果，会把这个结果放到task队列中。当主线程的执行栈中没有任务了，就回去task队列中取出异步任务的结果来执行。

这个过程不断循环，叫做event loop.

任务队列是一个事件的队列，IO设备完成一个任务（或者用户产生的操作，如鼠标点击，页面滚动，或者是settimeout），就在任务队列中添加一个事件，表示这个异步任务返回结果了，可以进入执行栈了。任务队列是先进先出的。任务队列分为microtasks和macrotasks

JavaScript 中有 microtasks 和 macrotasks（task），它们是异步任务的一种类型，Microtasks的优先级要高于macrotasks，macrotasks 用于处理 I/O 和计时器等事件，每次执行一个。microtask 为 async/await 和 Promise 实现延迟执行，并在每个 task 结束时执行。在每一个事件循环之前，microtask 队列总是被清空（执行）。

就是说，event loop是这样的：

  1. 先执行同步任务清空主线程的执行栈。如果执行栈是空的，跳过这一步。
  2. 然后清空microtask 队列。如果microtask 队列是空的，跳过这一步。
  3. 最后执行macrotask 队列中的一个任务。如果macrotask队列是空的，跳过这一步。
  4. 重复1-3步。

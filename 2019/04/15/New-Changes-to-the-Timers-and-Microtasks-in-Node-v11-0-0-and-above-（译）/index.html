Node v11 and above

使用 Node v11中的新更改，nextTick 回调和微任务将在每个单独的 setTimeout 和 setImmediate 回调之间运行，即使timer队列或immediates队列不是空的。

在 setTimeout 和 Promise 回调方面，Node v11中的新变化与浏览器行为相匹配，从而提高了 Node.js 中浏览器 JavaScript 的可重用性。

然而，这一重大变化可能会破坏现有的 Node.js 应用程序，因为它们显然依赖于旧的行为。 因此，如果您要升级到 Node v11或更高版本(最好是下一个 LTS v12) ，您可能需要考虑这一点。

就是说浏览器中的事件循环和Node v11 and above是一致的。nextTick 回调和微任务都是在单独的setTimeout 和 setImmediate 回调之间运行。

但是浏览器中的事件循环和Node v10是有区别的。Node v10 的nextTick 回调和微任务都是在setTimeout的timer queue 或者 setImmediate 的immediate queue 清空，执行下一个phase之前运行。
